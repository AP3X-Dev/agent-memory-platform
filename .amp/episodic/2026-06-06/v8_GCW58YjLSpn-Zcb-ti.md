Key findings worth remembering for future graph/MCP work in AMP:
- There is NO code-enforced "amp_store is the only write path" invariant. ~6 agent-facing MCP tools already write to Neo4j (amp_ingest, amp_arch_register/relate/aspect, amp_code_index, amp_braindump, amp_wiki_sync) and even the nominally read-only amp_arch_drift does SET e.stale (drift.ts:95,149,226). Persisting analytics nodes behind persist:true is precedented; just isolate them from amp_load/consolidation.
- Adding any new MCP tool/domain requires FIVE coordinated edits because ToolDomain (mcp/src/tools.ts:126-128) is a closed union typed Record<ToolDomain,string[]>: extend the union, add DOMAIN_TOOL_NAMES_MAP entry, add DOMAIN_DESCRIPTIONS entry, export *_TOOL_NAMES and append it in server.ts:454, and (if always-visible) ALWAYS_ON_TOOL_NAMES. The real drift guard is packages/mcp/src/__tests__/server.test.ts:72-89 (asserts registeredButUnlisted==[] && listedButUnregistered==[]), NOT tool-registration.regression.test.ts. Every tool needs a NON-EMPTY ToolAnnotations (ANN_WRITE/ANN_READONLY at tools.ts:1103-1112) or it re-triggers the resolved "typedHandler is not a function" SDK overload bug. Services are injected via module-level set*ServiceInstances() singletons in bootstrap.ts, not via registration args.
- GDS is in production docker-compose (NEO4J_PLUGINS includes graph-data-science) but NOT in CI (.github/workflows/ci.yml:61 = ["apoc"] only), so the GDS path ships untested by automation and the in-process fallback is the real primary path. neo4j/src/gds.ts is effectively dead code (only findSimilarSemantics calls a real GDS function gds.similarity.cosine; pageRank/communityDetection are pure-Cypher proxies with no non-test consumers).
- amp_compile's emit_graph is a dead flag (dropped by the bootstrap adapter; WikiCompiler.compile takes no such param) and only writes an unconditional Mermaid _graph.md (renderers.ts:763) -- there is no machine-readable graph export today, so amp_graph_export would be net-new.
- Secret-safety must be a per-node-type property allowlist at the snapshot boundary, not a labels-only rule: Symbol.signature/doc_comment, Semantic.content (incl. verbatim braindump text), embedding vectors, and absolute Source.path otherwise leak through the properties map into exports.
- Neo4j integers are Integer objects: all count/cohesion/centrality math needs toNum() coercion (lint.ts:88-91, symbol-store.ts:367) and LIMIT/SKIP need neo4j.int(). "Deterministic" requires explicit ORDER BY + ordered collect() + a final TS sort; Neo4j returns unordered without it.
- Build wiring: npm workspaces (not pnpm), @amp/<x> scope; new package needs composite/outDir:dist/rootDir:src and must be added to BOTH package.json workspaces AND tsconfig.build.json references (wiki is buggily missing from the latter). vitest runner uses --no-file-parallelism, not Jest's --runInBand; pin vitest ^3.2.4 (wiki's ^2.0.0 is the outlier). Stray src/*.js silently shadow .ts (not gitignored) -- never run bare tsc in src.
